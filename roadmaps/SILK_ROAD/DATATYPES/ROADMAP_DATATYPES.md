# ROADMAP: SILK_ROAD DATATYPES

## Scope
Deliver rich datatype correctness in incremental phases, starting with temporal and decimal foundation contracts, then expanding runtime coverage and optimization.

## Phase Plan

### Phase FOUNDATION
- Objective: Establish distinct TIMESTAMP and DATETIME contracts, timezone-aware semantics, and decimal coercion policy.
- Status: Planned.
- Discovery: [roadmaps/SILK_ROAD/DATATYPES/discovery/discovery-DATATYPES-FOUNDATION.md](roadmaps/SILK_ROAD/DATATYPES/discovery/discovery-DATATYPES-FOUNDATION.md)
- Implementation plan: [roadmaps/SILK_ROAD/DATATYPES/plans/plan-DATATYPES-FOUNDATION.md](roadmaps/SILK_ROAD/DATATYPES/plans/plan-DATATYPES-FOUNDATION.md)
- Completion matrix: [roadmaps/ROADMAP_DATATYPES_FOUNDATION_MATRIX.md](roadmaps/ROADMAP_DATATYPES_FOUNDATION_MATRIX.md)

### Phase EXPANSION
- Objective: Expand datatype compatibility for query operators and mutation paths beyond the FOUNDATION subset.
- Status: Not started.

### Phase OPTIMIZATION
- Objective: Use trusted datatype contracts for planner/runtime simplification, diagnostics, and operational safety.
- Status: Not started.

## Signoff Rule
Each phase requires a completion matrix with mandatory criteria marked Done and evidence links before phase signoff.
