# ROADMAP: SILK_ROAD UI

## Scope
Deliver a lightweight operational dashboard for kionas with a separate UI backend and web frontend, focused on real-time visibility for server statistics, sessions, tokens, and worker status. This foundation phase is observability-only and excludes authentication and control-plane mutation actions.

## Phase Plan

### Phase FOUNDATION
- Objective: Establish UI architecture and deliver a single auto-updating dashboard page backed by polling endpoints.
- Status: Planned.
- Discovery: [roadmaps/SILK_ROAD/UI/discovery/discovery-UI-FOUNDATION.md](roadmaps/SILK_ROAD/UI/discovery/discovery-UI-FOUNDATION.md)
- Implementation plan: [roadmaps/SILK_ROAD/UI/plans/plan-UI-FOUNDATION.md](roadmaps/SILK_ROAD/UI/plans/plan-UI-FOUNDATION.md)
- Completion matrix: [roadmaps/ROADMAP_UI_FOUNDATION_MATRIX.md](roadmaps/ROADMAP_UI_FOUNDATION_MATRIX.md)

### Phase HARDENING
- Objective: Improve resilience and operational quality (error handling, stale-data indicators, and pagination strategy).
- Status: Not started.

### Phase EXTENSION
- Objective: Expand beyond a single dashboard into deeper drill-down pages and optional control-plane actions.
- Status: Not started.

## Signoff Rule
Each phase requires a completion matrix with mandatory criteria marked Done and evidence links before phase signoff.
