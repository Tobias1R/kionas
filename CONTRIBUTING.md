# Contributing Guidelines

Thank you for your interest in contributing! To keep this project safe and maintainable, please follow these rules.

## 🚀 Workflow

1. **Fork or Branch**
   - Create a new branch for your work: `feature/your-feature-name` or `fix/bug-description`.

2. **Commit Messages**
   - Use clear, descriptive commit messages.
   - Example: `fix: correct typo in README` or `feat: add login form`.

3. **Pull Requests**
   - All changes must come through a Pull Request (PR).
   - PRs should describe the problem, solution, and any relevant context.
   - Draft PRs are welcome for early feedback.

4. **Reviews**
   - At least one review is required before merging.
   - Direct commits to `main` are **not allowed**.

## 🛡️ Code Quality

- Run tests locally before submitting a PR.
- Follow the project’s coding style and linting rules.
- Do not commit secrets, credentials, or environment files.

## 📚 Documentation

- Update relevant docs (`README.md`, code comments) when adding new features.
- If your change affects usage, include examples.

## 🔒 Security

- Report vulnerabilities privately to the maintainers.
- Do not open public issues for security concerns.
